# 8-Week Growth Loop (Solo)

Goal: reach 1k-3k stars with a repeatable build-distribute-feedback loop.

## Weekly operating cadence

- Monday: ship one product improvement with visible user value.
- Tuesday: polish docs/demo and prepare distribution assets.
- Wednesday: post short-form content (X/Reddit/掘金).
- Thursday: community day (issues, discussions, fixes).
- Friday: publish long-form post (breakdown/case study).
- Weekend: metrics review and next-week planning.

## Weekly deliverables

| Week | Product | Distribution | Community |
|---|---|---|---|
| 1 | README conversion rewrite | Intro post + 60s demo clip | Open Discussions board |
| 2 | 3 reproducible demos | Demo thread + command snippets | Add `good first issue` labels |
| 3 | Observability/cost stats | Technical breakdown article | Close first feedback loop PR |
| 4 | Reliability hardening | "before vs after" performance post | Triage SLA under 24h |
| 5 | Benchmark draft | Comparative post with charts | Invite contributors on roadmap |
| 6 | Template #1 (content) | Real user case post | Host async Q&A thread |
| 7 | Template #2 (code review) | Workflow tutorial | Merge first external contribution |
| 8 | Launch summary release | "8-week progress" launch post | Publish public roadmap v2 |

## Metrics to track every week

- Growth: stars, forks, watch.
- Conversion: README visits -> stars, demo runs.
- Quality: issue response time, test pass rate.
- Distribution: impressions, clicks, click->repo conversion.

## Operating rules

- Only one major product bet per week.
- Keep product and distribution effort around 1:1.
- If star growth stalls for 2 weeks, change narrative angle, not only features.
